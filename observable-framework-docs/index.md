**Observable Framework** is an [open-source](https://github.com/observablehq/framework) static site generator for data apps, dashboards, reports, and more. Framework includes a preview server for local development, and a command-line interface for automating builds & deploys.

You write simple [Markdown](https://observablehq.com/framework/markdown) pages — with interactive charts and inputs in [reactive JavaScript](https://observablehq.com/framework/javascript), and with data snapshots generated by [data loaders](https://observablehq.com/framework/data-loaders) in _any_ programming language (SQL, Python, R, and more) — and Framework compiles it into a static site with instant page loads for a great user experience. Since everything is just files, you can use your preferred editor and source control, write unit tests, share code with other apps, integrate with CI/CD, and host apps anywhere.

Framework includes thoughtfully-designed [themes](https://observablehq.com/framework/themes), [grids](https://observablehq.com/framework/markdown#grids), and [libraries](https://observablehq.com/framework/imports) to help you build displays of data that look great on any device, including [Observable Plot](https://observablehq.com/framework/lib/plot), [D3](https://observablehq.com/framework/lib/d3), [Mosaic](https://observablehq.com/framework/lib/mosaic), [Vega-Lite](https://observablehq.com/framework/lib/vega-lite), [Graphviz](https://observablehq.com/framework/lib/dot), [Mermaid](https://observablehq.com/framework/lib/mermaid), [Leaflet](https://observablehq.com/framework/lib/leaflet), [KaTeX](https://observablehq.com/framework/lib/tex), and myriad more. And for working with data in the client, there’s [DuckDB](https://observablehq.com/framework/lib/duckdb), [Arquero](https://observablehq.com/framework/lib/arquero), [SQLite](https://observablehq.com/framework/lib/sqlite), and more, too.
